When talking about design patterns object-oriented programming (OOP) and functional programming (FP) are considered to be the opposites. FP is characterised by lack of state and immutable data structures while OOP is written using mutable data structures that represent state in programs. Even though Python is optimized for writing imperative programs (i.e., OOP) it works fairly well for FP. It is mainly because Python supports the concept of first-class functions. If you are on Ubuntu, you can get QEMU using aptitude

The GNU Debugger which is probably on your system doesn’t support obscure architectures like MIPS. You’ll need gdb-multiarch for that. It can also be installed without any hassle.
