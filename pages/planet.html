Wheels are faster, even for pure Python packages

When installing with pip (or from PyPI in general), wheels are much faster than source distributions, even for pure-Python projects. Packages with native code are a clearer win, because the wheel file will contain pre-compiled binaries for the platform you’re installing on. This means that you don’t need to have a compiler and non-Python build dependencies installed, and you don’t need to wait for the compiler to do its thing.

Intel oneAPI icpx Windows

Intel oneAPI 2023.0.0 newly provides GNU-like (icpx) and MSVC-like (icx) compiler drivers for Linux and Windows. This is a good thing, but CMake had assumptions that only C++ compiler driver icpx OR icx would be present. We created a CMake fix. Long term, GNU-like support for Windows may also come. Until you update CMake, you can workaround this issue of C++ oneAPI compiler detection on Windows by configuring the CMake project like:

cmake -Bbuild -DCMAKE_CXX_COMPILER=icx
